Detailed Syllabus

Software Architecture via Games

Games are complex software. By building them, students learn how to structure large codebases using Classes, Objects, and Inheritance.

1

The Game Engine (Pygame)

Weeks 1-4

We peel back the layers of how software renders graphics. Students learn about the "Game Loop"‚Äîthe infinite cycle that updates logic and draws frames 60 times per second.

The Game Loop: Event handling, Updating, and Drawing.
Coordinate Systems: X/Y axes and Screen Surfaces.
Input Handling: Managing complex keyboard and mouse states.
2

Object-Oriented Programming (OOP)

Weeks 5-8

The most critical skill for modern software engineers. Instead of messy code, we use Classes to create blueprints for Players, Enemies, and Bullets, making code scalable and reusable.

Classes & Objects: Instantiating multiple enemies from one blueprint.
Inheritance: Creating a "Boss Enemy" that inherits traits from a normal "Enemy".
Sprite Groups: Managing hundreds of objects efficiently.
3

Physics & Mechanics

Weeks 9-11

Making it feel real. We apply math (Vectors) to simulate gravity, friction, and movement. We also implement Collision Detection‚Äîthe math behind knowing when a laser hits an alien.

Collision Logic: Hitboxes (Rects) and overlap testing.
Physics: Velocity, Acceleration, and Friction.
Sound Engines: Triggering effects and background music.
4

Capstone & Polish

Weeks 12-14

The difference between a demo and a product. Students implement Game States (Start Menu, Pause, Game Over), High Score persistence using files, and compile their game into an executable (.exe).

Game States: Finite State Machines for menu logic.
File I/O: Saving and loading High Scores.
Exporting: Converting Python scripts to standalone Apps.
